# Thesis Repository – Career Transition Prediction

This repository contains all code, data processing steps, models, and experiments for the thesis on career transition prediction using SBERT and ESCO ontology features.

---

## Data & Resources

- **ESCO dataset (v1.2.1)**  
  Available in `Data/ESCO Files/` or downloadable from:  
  https://esco.ec.europa.eu/en/use-esco/download

- **JobHop dataset**  
  https://huggingface.co/datasets/aida-ugent/JobHop  
  (requires Hugging Face authentication)

- **Fine-tuned SBERT model**  
  https://drive.google.com/drive/folders/1xA7B6MzZtXPAW0wnanCesZ3MntibVVaM?usp=share_link

---

## Repository Structure

- `Data/`
  - `ESCO Files/` → ESCO raw data  
  - `Dataset/` → train/validation/test splits (Notebook 01, not stored in repo)  
  - `Ontology Artifacts/` → ESCO-derived features (Notebook 04)  
  - `TopK/` → candidate sets (Notebook 05, not stored in repo)

- `Models/` → trained SBERT models (not stored in repo, available through Google Drive link)

- `Results/`
  - `Baseline/` → SBERT results  
  - `Hybrid/` → hybrid results  
  - `Rerankers/` → reranker results  

- `Utils/`
  - `thesis_utility.py` → shared helper functions used across notebooks

---

## Notebooks Overview

### **Notebook 01 – Dataset Construction**
- Loads ESCO + JobHop datasets  
- Produces train/validation/test splits  
- **Output:** `Data/Dataset/`  
- Must be run (not stored in repo)

---

### **Notebook 02 – SBERT Training**
- Loads dataset from Notebook 01  
- Trains SBERT bi-encoder  
- **Output:** trained model  
- Recommended to run on GPU (e.g., Google Colab)
- Available through Google Drive link

---

### **Notebook 03 – Baseline Evaluation**
- Loads dataset (Notebook 01)  
- Loads fine-tuned SBERT  
- Evaluates:
  - off-the-shelf SBERT  
  - fine-tuned SBERT  
- **Output:** results in `Results/Baseline/` (already saved)

---

### **Notebook 04 – ESCO Feature Construction**
- Loads ESCO data  
- Builds ontology-based features (skills, IDF, hierarchy)  
- **Output:** `Data/Ontology Artifacts/` (already saved)

---

### **Notebook 05 – Hybrid Retrieval**
- Loads:
  - dataset (Notebook 01)  
  - SBERT model  
  - ESCO data + ontology artifacts (Notebook 04)  
- Computes:
  - top-K candidates  
  - hybrid scores (fixed + tuned)  
- **Outputs:**
  - results in `Results/Hybrid/` (already saved)  
  - top-K features in `Data/TopK/` (must be generated, not stored in repo)

---

### **Notebook 06 – Reranking Models**
- Loads top-K data (Notebook 05)  
- Trains:
  - Logistic Regression  
  - XGBoost reranker  
- **Outputs:**
  - results in `Results/Rerankers/` (already saved)  
  - reranker scores (must be generated, not stored in repo)

---

### **Notebook 07 – Results Analysis**
- Loads all evaluation results  
- Compares baseline, hybrid, and reranking methods  

---

### **Notebook 08 – Feature Analysis (EDA)**
- Loads:
  - top-K validation data (Notebook 05)  
  - tuning results  
  - reranker outputs (Notebook 06)  
- Performs feature-level analysis and diagnostics  

---

## Notes

- Some artifacts (datasets, models, top-K files) are not included due to size and must be generated by running the corresponding notebooks.
- All paths are relative; no absolute paths are required.
- GPU is recommended for SBERT training (Notebook 02).
